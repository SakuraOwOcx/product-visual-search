05

Why articleType

The project treats product type as the practical definition of similarity.

Why this label works

articleType gives a useful category-level answer to "is this retrieval relevant?" It is more concrete than broad categories and more stable than noisy free-form names.

Example 1Tshirts should retrieve Tshirts
Example 2Watches should retrieve Watches
Example 3Handbags should retrieve Handbags

What it does not capture

This definition is still coarse. Two items may share the same articleType while differing in color, style, or fine-grained visual details.

Category-level Not style-level Not user-feedback based

Overall pipeline

From raw metadata to measurable retrieval performance.

01

Data preparation

Load styles.csv, match rows with image files, filter invalid examples, remove rare classes, and create train / validation / test splits.

02

Model route execution

Train supervised ResNet18, extract frozen CLIP embeddings, and fine-tune CLIP using articleType supervision.

03

Embedding index construction

Encode gallery images into normalized vectors and save embeddings together with product metadata.

04

Retrieval evaluation

Encode test queries, compare them against the gallery, and measure Recall@1, Recall@5, and Recall@10.

  • src/product_search/index_utils.py saves and loads retrieval indexes.
  • src/product_search/search_engine.py performs similarity search.

  • prepare_full_dataset_split.py prepares the final split.

Evaluation metrics

The project measures both classification quality and retrieval quality.

Classification

Top-1 accuracy

Checks whether the model predicts the correct articleType as the single best class.

Classification

Top-5 accuracy

Checks whether the correct class appears anywhere in the top five predicted classes.

Retrieval

Recall@1

Asks whether the first returned product already matches the query's articleType.

Retrieval

Recall@5 / Recall@10

Ask whether at least one relevant result appears inside a broader top-k candidate set.

In the final run, the train split is used as the gallery and the test split is used as the query set.

Overall comparison logic

There is no single universal winner; the answer depends on the use case.

Best first resultResNet18 supervised
Best broad top-k recallFrozen CLIP ViT-B/32
Best adaptation evidenceFine-tuned CLIP ViT-B/32

ResNet18 is best when the first returned item must already be correct. Frozen CLIP is best when we care more about broad candidate coverage.

19

Limitations and future work

The system is complete, but it is still category-level and offline.

Current limitations

  • Retrieval correctness is mainly defined by articleType, not fine-grained visual style.
  • Some classes are long-tailed and have much fewer examples than others.
  • The system is evaluated offline and does not include real user feedback signals.
  • The current index uses direct embedding similarity instead of a scalable ANN backend.

Future work

  • Use richer labels such as type + color + gender + usage.
  • Try metric-learning losses like contrastive loss or triplet loss.
  • Add text-query or image-text hybrid retrieval with CLIP.
  • Introduce approximate nearest-neighbor indexing and metadata reranking.
